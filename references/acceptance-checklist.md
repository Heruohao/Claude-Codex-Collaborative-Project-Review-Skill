# Acceptance Checklist

Use this checklist after implementation and before closing a project.

## Acceptance Inputs

- Approved plan:
- Final scope:
- Delivered files, links, documents, or deployed outputs:
- Acceptance criteria:
- Verification evidence:
- Known limitations:

## Delivery Verification

- The delivered work matches the approved scope.
- All required deliverables are present.
- Acceptance criteria have direct evidence.
- Tests, previews, screenshots, documents, or logs are available when useful.
- Unfinished items are clearly labeled.
- No unrelated project files or assumptions were mixed in.

## Claude Joint Acceptance

Use this section when the user says “启动 Claude 联合验收” or when an important delivery needs external acceptance review.

- Prepare a compact acceptance packet for Claude.
- Include the approved plan, completed work summary, acceptance criteria, evidence, and known limitations.
- Ask Claude whether each acceptance criterion passes, fails, or needs more evidence.
- Ask Claude to identify missing deliverables, hidden risks, and recommended fixes before acceptance.
- If Claude CLI cannot be used, provide a ready-to-copy prompt for Claude App.
- Compare Claude's findings with Codex's own verification and the user's decision standard.

## Acceptance Result

End with one of:

- **Accepted**: all material criteria passed.
- **Conditionally accepted**: usable, but named fixes or follow-ups remain.
- **Not accepted**: material criteria failed or evidence is insufficient.

The user makes the final acceptance decision.
