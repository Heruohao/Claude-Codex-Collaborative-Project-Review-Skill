# Comparison Framework

Use this framework when similar projects, products, cases, or implementation models exist.

## Comparable Selection

Choose comparables that match at least two of:

- same user group
- same problem
- same delivery channel
- same legal or operational environment
- similar cost or complexity
- similar success metric

Avoid weak comparisons that only share surface features.

## Evaluation Dimensions

- user value
- implementation complexity
- cost and maintenance
- legal/compliance exposure
- scalability
- content or data requirements
- operational burden
- monetization or impact potential
- time to first usable version

## Decision Recommendation

End with one of these recommendations:

- **Proceed**: benefits justify implementation under the proposed scope.
- **Proceed with reduced scope**: the idea is valid, but the first version should be smaller.
- **Pause for research**: key facts are missing.
- **Revise concept**: the original direction is weak but can be reshaped.
- **Reject**: risks, cost, or weak value outweigh expected benefits.

Always explain what should be copied, changed, and avoided from comparable examples.
